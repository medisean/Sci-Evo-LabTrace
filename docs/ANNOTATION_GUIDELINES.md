# Sci-Evo-LabTrace 标注规范

## 目标

本规范用于保证 gold case 不是论文摘要，而是可训练、可评测、可追溯的科研演化轨迹。每条 case 必须记录科研问题如何被拆解、决策如何发生、工具和实验如何被选择、观察结果如何推动下一步，以及最终如何验证成功。

## 合格 Gold Case 标准

一条 gold case 至少满足以下要求：

1. 有明确科研目标和量化成功指标。
2. 至少包含 4 个科研轨迹步骤。
3. 每一步包含 `thought`、`action`、`tool`、`parameters`、`observation` 和 `evidence`。
4. 轨迹中必须体现“已知背景、未解决缺口、下一步决策”的因果关系。
5. 至少包含一次计算/分析步骤和一次实验/验证步骤，除非论文领域本身不包含湿实验。
6. 最终验证必须包含可量化指标或明确评价标准。
7. 来源许可状态必须写入 `source.license_status`。
8. 公开发布前必须确认 PDF、图片、表格和 MinerU 全量解析产物的许可边界。

## Evidence 规范

`evidence` 字段不是普通引用，而是结构化溯源信息。每条 evidence 至少包含：

- `source_doc`：来源文档或 URL。
- `page`：证据所在页码。
- `locator`：章节、图表、段落或方法位置。
- `support`：该证据支持的数据字段或科学判断。

推荐在 MinerU 解析产物可用时补充 `mineru_block_id`，将字段级证据进一步绑定到 content list 或 layout block。

## 轨迹步骤拆分原则

步骤不应按论文段落机械切分，而应按科研决策切分：

- 一个工具/实验选择对应一个步骤。
- 一个观察结果导致新决策时，应开启新步骤。
- 如果论文出现失败、低活性、模型不匹配或结构偏差，必须保留为 gap 或 partial outcome。
- 若一个步骤只描述结论、没有决策或观察，不应单独成为 Sci-Evo step。

## 质量分级

- `gold`：人工复核，证据完整，字段稳定，可直接用于评测或训练。
- `silver`：自动抽取或半自动整理，字段基本完整，但仍需人工复核。

当前提交只公开 gold case，避免用低质量 silver 数据稀释评审印象。

## 本项目当前口径

当前 3 条 gold case 覆盖同一主线：AI/ML 驱动的蛋白与酶工程科研演化。这样可以保持主题聚焦，同时展示从头设计、机器学习文库设计、定向进化复盘和 ensemble-based 计算再设计等不同演化模式。
